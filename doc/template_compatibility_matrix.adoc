= Template Compatibility Matrix
:icons: font

This document compares the Facelets template contract exposed by the upstream
AdminFaces `admin-template` with the contract currently exposed by
`pf-adm-template`.

The focus here is template composition compatibility:

* `ui:define` regions exposed by `admin.xhtml`
* `ui:insert` regions exposed by `admin-base.xhtml`
* whether a hook exists, not whether the full AdminFaces behavior behind that hook is implemented

IMPORTANT: `pf-adm-template` currently aims for structural compatibility of
template regions. It does not currently implement the AdminFaces control sidebar
feature set or all legacy AdminFaces backing-bean APIs.

== Legend

* icon:check[font-color: green] = available
* icon:times[font-color: red] = not available
* icon:warning[font-color: orange] = available as a hook, but not backed by the full AdminFaces feature set

== Summary

`pf-adm-template` now exposes the core Facelets composition regions needed for
AdminFaces-style page templates:

* header and menu wrapper regions are available
* the main content regions are available
* menu and header extension points are available

The remaining compatibility gap is not the existence of hooks, but feature
behavior behind some of them, especially the AdminFaces control sidebar.
