= Configuration Property Compatibility Matrix
:icons: font

This document compares the configuration surface of the upstream AdminFaces
`admin-template` with the Spring Boot configuration properties currently exposed
by `pf-adm-template`.

`pf-adm-template` configuration is provided by `PfAdmProperties` with the prefix
`pf-adm`.

The matrix focuses on property availability, not on full behavioral parity.

== Legend

* icon:check[font-color: green] = supported
* icon:check[font-color: yellow] = partially supported or supported with different behavior
* icon:times[font-color: red] = currently not supported

== Summary

`pf-adm-template` currently supports the core shared configuration needed for
template selection, messaging, breadcrumbs, sidebar usage, and skin selection.

The main AdminFaces compatibility gap is the larger behavioral configuration
surface around ripple effects, ajax/loading UX, control sidebar behavior,
top-menu layout, and other AdminFaces-specific interaction features.
