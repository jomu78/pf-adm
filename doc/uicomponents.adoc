= UI Component Compatibility Matrix
:icons: font

This document compares the visual/component-level support offered by the
upstream AdminFaces `admin-template` and `pf-adm-template`.

The table is intentionally high level. It answers this question:

* does the template/theme provide styling and layout support for the component family?

It does not guarantee identical markup, JavaScript behavior, or pixel-perfect
rendering across both projects.

== Summary

`pf-adm-template` currently has the strongest compatibility in core layout and
shared presentation areas such as panels, buttons, menus, data tables, tab
views, dialogs, and date pickers. Most specialized PrimeFaces component
families still need dedicated compatibility styling before they can be
considered AdminFaces-equivalent.
