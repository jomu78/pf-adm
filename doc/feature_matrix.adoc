= Feature Matrix
:icons: font

This document compares higher-level template features between the upstream
AdminFaces `admin-template` and `pf-adm-template`.

Unlike the template compatibility matrix, this page is about implemented
behavior, not just the existence of Facelets hook names.

* icon:check[font-color: green] = implemented
* icon:check[font-color: yellow] = partially implemented or implemented differently

== Summary

`pf-adm-template` already matches AdminFaces in core areas such as responsive
layout, sidebar layout, error pages, configuration support, Font Awesome
integration, and skin variants.

The main missing areas are AdminFaces-specific UX features:

* control sidebar customization
* top-menu layout variant
* menu search
* ripple/material interactions
* breadcrumb navigation behavior beyond basic rendering
